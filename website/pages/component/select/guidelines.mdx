Select inputs are typically used alongside other form elements like toggles, radio boxes, or text inputs when a user needs to make a selection from a list of items.

<div
  className={css`
    margin: 40px 0px;
    width: auto;
    height: 208px;
    background-image: url('${CDN}/images/guidelines/LM-selectInputs-1@3x.png');
    background-position: center center;
    background-repeat: no-repeat;
    background-size: contain;
  `}
/>

Select inputs are not menus. In a menu, a user takes an action (like copy/delete a property or navigate to a different section/screen). In a select input, a selection is being made — the user’s selection will be shown after a selection is made (unlike with the menu).

  `}
/>

Unlike buttons, there is no primary or secondary select input. Use a progressive disclosure pattern so as to emphasize the input the user should interact with before surfacing other UI elements. (For example, for a billing form, show the country select input before surfacing the state select input, since not all countries have states.)
