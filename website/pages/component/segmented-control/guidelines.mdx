# When to use segmented controls

## Use segmented controls to toggle views within a page

For example, to switch between code and table views or filter data.

<GuidelineImage
  imageURL={`${imagePath}/SC-guidelines-GIF-1.gif`}
  className={usageImageStyle}
/>

## Use segmented controls as filter selects

Segmented controls can sometimes act as toggles to encourage comparison between types of options, for example annual and monthly. However, they are not a replacement for radio boxes in other use cases.

<GuidelineImageFrame type="do">
  <img src={`${imagePath}/SC-guidelines-GIF-2.gif`} className={gifStyle} />
</GuidelineImageFrame>

## Use segmented controls as filter selects

Segmented controls are often used to toggle through view types within a single page. Consider using <Link href="component/tabs/guidelines">tabs</Link> to flip between distinct pages.

<GuidelineImage
  imageURL={`${imagePath}/SC-guidelines-GIF-3.gif`}
  className={usageImageStyle}
/>

## How to use segmented controls

Segmented controls should contain between two and five equal-width segments. The width of each segment should be equal to the width of the largest segment.

<GuidelineImage
  imageURL={`${imagePath}/SC-guidelines-4@3x.png`}
  className={detailImageStyle}
/>

Exactly one segment is always selected. If a state is higher priority, consider making it the default selection.

<GuidelineImage
  imageURL={`${imagePath}/SC-guidelines-5@3x.png`}
  className={detailImageStyle}
/>

Use an external label to avoid including unecessary words into the segments.
