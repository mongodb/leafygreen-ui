Toasts are a form of notification. They should be used fairly sparingly, since they animate and can become overwhelming to users if used to confirm every action.

Below is a table to help you choose when to pick which pattern for notifications.

Toasts can either be dismissable (via the top "X"), or can disappear automatically after a set amount of time — to figure out this time, factor for a reading speed of 150 words per minute, plus allow for another second or so to allow for enough time for the user to read the toast content).

Do not use toasts for notifying a user of a local action (like pressing copy on a copyable component). Toasts should remain visible regardless of the page the user is on until they are dismissed.

Toasts can also be used to show status of longer ongoing processes (lasting more than 4-5 seconds). When doing so, try to add an estimate of the time the process will take to complete, or where in the process the user is.

Do not nest actions within toasts. Our progress toast allows for an action to be contained within it, such as for viewing the current progress of a cluster rebuild or pausing the current process.
